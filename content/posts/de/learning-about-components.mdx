## Markdown, aber besser

Hallo, das aktuelle Datum ist <DateTime format="local" />. Markdown ist großartig zum Schreiben von Inhalten, aber was wenn Sie etwas Interaktiveres benötigen? Was wenn Sie dynamische Daten abrufen, formatierte Callouts anzeigen oder Benutzereingaben sammeln möchten? Hier kommen Komponenten ins Spiel.

***

```graphql
query MyQuery($relativePath: String!) {
  page(relativePath: $relativePath) {
    title
  }
}
```

> Mit Komponenten ist Ihr Markdown-Inhalt nicht nur statischer Text—er wird zu einem dynamischen Erlebnis. Sie können Daten abrufen, Inhalte schön formatieren und sogar interaktive Elemente direkt in Ihren Beiträgen erstellen.

## Warum Komponenten in Markdown verwenden?

* **Inhalte dynamisch halten** - Echtzeitdaten statt statischen Text abrufen.
* **Lesbarkeit verbessern** - Strukturierte Callouts, Buttons und Formulare verwenden.
* **Benutzerinteraktion verbessern** - Lassen Sie Benutzer direkt im Inhalt handeln.

## Weitere Möglichkeiten, Komponenten zu verwenden

1. **GraphQL-Abfragen einbetten** - Inhalte dynamisch abrufen und anzeigen.
2. **Gestylte Callouts verwenden** - Wichtige Punkte mit visuell unterscheidbaren Elementen hervorheben.
3. **Interaktive Formulare erstellen** - Lassen Sie Benutzer sich anmelden, Daten übermitteln oder mit Ihrem Inhalt interagieren.
